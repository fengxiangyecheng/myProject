# 角色模块回归检查清单

本文用于按“角色 -> 模块 -> 用例”做验收回归，目标是快速发现权限、数据域、字段映射、交互联动问题。

## 0. 环境前置

- 执行数据库补丁：
  - `db/patch_app_user_account_status.sql`
  - `db/patch_app_user_user_prefs.sql`
  - `db/patch_personal_role_menus.sql`
  - 建议同步执行：`upgrade_requirements_v2.sql`
- 准备账号：
  - 系统管理员（`role_manage_id=1`）
  - 团队管理员（`role_manage_id=2`）
  - 个人用户（`app_user` 登录，`login_user_type=personal`）
- 清理浏览器缓存并重新登录（保证 `localStorage.userInfo` 为新 session）。

## 1. 系统管理员（role=1）

- 用户管理
  - 禁用/启用 `app_user` 后，列表状态即时刷新。
  - 禁用用户不可登录，提示“账号已被禁用”。
- 角色/菜单/权限
  - 从左侧菜单进入各页，新增/编辑/删除/查询按钮与权限一致。
  - 从首页卡片快捷入口进入页面，按钮权限不应错乱（应与左侧菜单一致）。
- 日程类型管理
  - 新增类型编码唯一；停用后任务页类型下拉不再展示该项。
- 操作日志
  - 登录成功/失败/禁用拦截应产生日志。

## 2. 团队管理员（role=2）

- 团队管理数据域
  - 仅能看到：自己创建、自己管理、自己加入的团队。
  - 不应看到无关团队。
- 团队成员
  - 仅可维护有权限的团队成员（新增/删除/修改团队角色）。
  - 非授权团队应提示无权限。
- 团队日程/任务
  - 分配人必须是启用中的系统管理员/团队管理员（1/2）。
  - 被分配人必须是启用中的个人用户（`app_user`）。

## 3. 个人用户（app_user）

- 登录与个人中心
  - 登录后进入 `idx_center` 显示个人设置表单（不是管理员表单）。
  - 保存成功后，刷新仍保留偏好（`user_prefs`）。
- 默认视图联动
  - 个人中心设置 `defaultView` 为：
    - 月视图 -> 任务页默认本月筛选
    - 周视图 -> 任务页默认本周筛选
    - 日视图 -> 任务页默认今日筛选
    - 列表 -> 不加日期筛选
- 列显隐联动
  - 个人中心勾选任务列表字段并保存，任务页列显示应对应变化。
- 收藏
  - 点击“收藏/已收藏”可切换，刷新后状态保持。
- 个人统计
  - 完成率、类型分布、提醒达标率可返回且与任务数据一致。

## 4. 安全与边界

- 仪表盘越权检查
  - 手工构造 `queryDashboardData?user_id=其他人`，返回仍应是当前登录用户数据。
- 空值/类型检查
  - 编辑任务时切换被分配人，不能因旧 `assignee_id` 残留导致“被分配人无效”。
- 错误可观测性
  - 个人中心保存失败时，前端要有错误提示（非静默失败）。

## 5. 已知未完全覆盖（任务书差距）

- 邮件/系统消息真实发送链路（SMTP + 定时任务）尚未完整落地；
  当前已完成偏好存储（`user_prefs`），可作为后续提醒引擎输入。
- `sys_system_config` 仅有建表与初始数据，后台管理页面与接口需补齐。
